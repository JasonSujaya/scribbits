## What this app is

Scribbits Arena is a portrait Reddit game where each player draws one creature, called a Scribbit, and the drawing's shape becomes its combat build. The server analyzes submissions, owns progression and outcomes, and returns deterministic battle reports; the client presents those reports as lively real-time-looking replays. Each Scribbit has a three-day life that can end as a permanent Legacy Card, while daily Rumbles, Champion fights, care, sparring, cosmetics, and scouting create reasons to return. (`app/src/server/routes/api.ts`, `app/src/client/game.ts`)

## Why it's valuable

The hook is immediate and personal: a player's own drawing visibly changes how a fight unfolds, but cannot buy raw stat power. The critical path is **draw a Scribbit → submit it → watch that exact drawing fight → understand the result**; if any link breaks, the game loses its reason to exist. (`app/src/client/scenes/Draw.ts`, `app/src/server/core/battle.ts`)

## The main flows

1. **A player opens the Reddit post → enters the expanded game → chooses the timed Community Theme or untimed Free Draw.** The Community Theme creates the daily Scribbit, brings it alive, and immediately pits that exact drawing against one simple server-selected founding NPC; the result then returns to Home. Free Draw uses the same canvas but saves separately without a Rumble entry, battle, or reward. Returning to Draw on the same Arena day shows that verified saved image and one Practice action. The server rechecks every PNG and owns both write paths. (`app/src/client/splash.ts`, `app/src/client/scenes/ScribbitHome.ts`, `app/src/client/scenes/Draw.ts`, `app/src/server/routes/api.ts`, `app/src/server/core/freeDrawingStore.ts`)
2. **The server resolves a fight → the player watches a deterministic replay → the result returns to the right scene.** The fixed-tick engine owns outcomes and the arena micro-goal score; Replay only projects the immutable timeline, reports that stored goal state, and stages follow-up actions through the registry. A signed-in player may explicitly share a silent recording of the rendered canvas through Reddit media, but that presentation clip is not authoritative battle proof. (`app/src/shared/combat/engine.ts`, `app/src/shared/battlearena.ts`, `app/src/client/scenes/Replay.ts`, `app/src/client/lib/battleclip.ts`)
3. **A returning player opens Home → sees their living Scribbit → taps one large Draw button when they want to create or practice.** Home is the centered primary dock destination and intentionally does not show care, detail, or battle controls; Arena remains the fighting hub. (`app/src/client/scenes/ScribbitHome.ts`, `app/src/client/lib/appdock.ts`, `app/src/client/scenes/Draw.ts`)
4. **A returning player opens Arena → sees today's venue and goal → chooses one owned fighter → chooses Champion or Spar.** Champion starts its one daily contract. Spar first opens three server-ranked `SAFE`, `EVEN`, and `BOLD` rivals; the player chooses the risk and points before the authoritative battle begins, then continues the same three-bout Rival Run from each result. Rival Run challenge cards can now ask the player to trigger their drawing's Shape Power; the server counts only that fighter's authoritative `ability_activated` transcript events. Arena is the one home for starting living-Scribbit battles; Battles remains replay history, Draw owns creation and Practice, and the generic Scribbit detail modal owns care/inspection rather than combat initiation. (`app/src/shared/battlearena.ts`, `app/src/shared/rivalrunchallenges.ts`, `app/src/server/core/rivalRun.ts`, `app/src/client/scenes/ArenaHome.ts`, `app/src/client/lib/rivalrunflow.ts`, `app/src/client/scenes/MyBattles.ts`)
5. **A player opens Arena → makes one Rumble Pick → the server locks that daily prediction → the nightly job awards Clout and files a receipt.** Rumble Pick is a compact secondary Arena action, not a fight mode; the client never decides eligibility, placement, or payout. (`app/src/client/scenes/ArenaHome.ts`, `app/src/server/core/clout.ts`, `app/src/server/core/dailyJob.ts`)
6. **A player finishes the first Rival Run → follows `FIRST GEAR` through only the Care still needed → opens a simplified first chest → lands in Bag on the reward's equipment category.** The server refreshes the exact Ink and Care state before each step, and the first chest always grants equippable Gear while preserving the published rarity odds. Later Shop visits support one or ten Mystery Ink Chests; ten is the largest batch, Epic pity lands by ten, duplicate copies remain Forge material, and the server persists every open and loadout. (`app/src/client/lib/firstchesttrail.ts`, `app/src/client/scenes/Replay.ts`, `app/src/client/scenes/ArenaHome.ts`, `app/src/client/scenes/Shop.ts`, `app/src/client/lib/capsulemachine.ts`, `app/src/server/core/inkStore.ts`)
7. **A Scribbit reaches the end of its life → the server freezes a Legacy Card → the player opens Gallery from Home.** Gallery owns public Legends and personal Legacy; Bag owns mutable inventory and equipment. Home keeps Gallery directly available without displacing a primary dock destination; Settings owns the Field Guide. (`app/src/shared/legacycards.ts`, `app/src/client/scenes/ScribbitHome.ts`, `app/src/client/lib/appmenu.ts`, `app/src/client/scenes/Gallery.ts`)

## Expected user behavior

Players draw in portrait from Home's large Draw button, open Gallery directly from Home for Legends and Legacy, watch server-authored fights, make one meaningful daily Pick from Arena, care for living Scribbits where existing battle/detail flows expose care, open Shop for earned-Ink Mystery Chests, and open Bag for equipment. The primary dock is Arena, Bag, Home, Battles, and Shop, with Home centered and Draw intentionally removed from the dock. The Shop uses a generated scrapbook stage, matched closed/open chest art, and a generated Ink token behind code-owned motion, controls, odds, and Gear previews. The only purchase sizes are `×1` and `×10`; ten rewards reveal in their paid order and remain visible together, with no 100-open or automatic repeat. Arena shows the selected Scribbit and opponent as one matchup, then offers one primary Fight or rival-choice action; the compact Rumble control remains secondary. Discovered Gear is a reusable account unlock that may be equipped on multiple living Scribbits; loose copies are Forge material, not a second ownership requirement. (`app/src/client/assets/scribbits-home-stage.png`, `app/src/client/assets/scribbits-shop-stage.png`, `app/src/client/assets/scribbits-shop-chest-closed.png`, `app/src/client/assets/scribbits-shop-chest-open.png`, `app/src/client/assets/scribbits-ink-token.png`, `app/src/client/lib/capsulemachine.ts`, `app/src/client/lib/capsulereveal.ts`)

## Not built yet

- There is no synchronous networked PvP transport; combat is server-resolved and then streamed from an immutable report as a replay. (`app/src/server/core/battle.ts`, `app/src/client/scenes/Replay.ts`)
- Earned Gear resolves into at most one bounded technique per category in Exhibition/Spar, with the strongest piece leading and the second supporting. It is snapshotted into deterministic v3 combat without changing the fixed 100-point drawing build. Rumble, Champion, and Practice remain Gear-neutral by design. Exhaustive mirrored tests cover all six Gear effect families and both equipment slots. Birth-time accessories remain a separate consumed, baked-image system. (`app/src/shared/gearcombat.ts`, `app/src/server/core/battle.ts`, `app/src/server/core/submission.ts`, `app/tests/gear-combat-balance.test.mjs`)
- Scout Notebook is no longer a primary navigation destination. Its scene and API remain temporarily for saved-replay compatibility and can be removed after those return paths no longer reference it. (`app/src/client/scenes/ScoutNotebook.ts`, `app/src/server/core/scoutNotebook.ts`)
- The disabled `Reddit Gold Styles` banner is only a coming-soon preview; no product, entitlement, paid randomization, or fulfillment route exists. (`app/src/client/lib/capsulemachine.ts`)
